package.json

Why?

It acts as the blueprint of the project by storing project metadata, dependencies, and scripts needed to run the application.

package-lock.json

Why?

It locks the exact versions of all installed packages so everyone gets the same environment and avoids version conflicts.

node_modules

Why?

It is the local folder where npm downloads and stores all the packages and their dependencies required for the project to run.
